# เกณฑ์การประเมินผลโครงงาน (Project Assessment Rubrics)

โครงการนี้มีน้ำหนักคะแนนรวม 40% ของทั้งรายวิชา โดยแบ่งการประเมินตาม Milestone ดังนี้

---

### **Milestone 1: Foundations & High-Level Design (10%)**
**เป้าหมาย:** วัดความเข้าใจในปัญหา, การวิเคราะห์ความต้องการ, และการออกแบบระดับแนวคิด

**1. คุณภาพของการวิเคราะห์ความต้องการ (40%)**
*   **ยอดเยี่ยม (9-10):** Use Cases มีคุณค่าต่อ Actor ชัดเจน ไม่ใช่แค่ฟังก์ชัน, Description แบบ "Fully Dressed" มีรายละเอียดลึกซึ้ง ครอบคลุมทุก Flow
*   **ดี (7-8):** Use Cases ส่วนใหญ่ถูกต้อง, Main Flow ชัดเจน แต่ Alternative Flows อาจขาดรายละเอียด
*   **พอใช้ (5-6):** ระบุ Use Cases ได้ แต่อาจยังสับสนกับฟังก์ชัน, Description มีแค่ Main Flow
*   **ต้องปรับปรุง (0-4):** ระบุ Use Cases ผิดพลาด, ไม่มี Description ที่มีความหมาย

**2. คุณภาพของการออกแบบระดับแนวคิด (30%)**
*   **ยอดเยี่ยม (9-10):** Conceptual Model (S.O.M.) สะท้อน Domain ได้ถูกต้อง, Quality Attribute Scenarios มีความเฉพาะเจาะจงและวัดผลได้
*   **ดี (7-8):** Conceptual Model ส่วนใหญ่ถูกต้อง, QAs ที่ระบุมายังค่อนข้างกว้าง
*   **พอใช้ (5-6):** มี Conceptual Model แต่ยังขาดความสมบูรณ์หรือมีข้อผิดพลาดสำคัญ
*   **ต้องปรับปรุง (0-4):** ไม่มี Conceptual Model หรือไม่สอดคล้องกับ Use Cases

**3. การนำเสนอและตอบคำถาม (30%)**
*   **ยอดเยี่ยม (9-10):** นำเสนอเป็นมืออาชีพ, แสดงถึงความเข้าใจที่เป็นหนึ่งเดียวกันของทีม, ตอบคำถามและปกป้องการตัดสินใจได้อย่างมีเหตุผล
*   **ดี (7-8):** นำเสนอชัดเจน, สมาชิกส่วนใหญ่ตอบคำถามได้
*   **พอใช้ (5-6):** การนำเสนอวกวน หรือมีเพียงคนเดียวที่ตอบคำถามได้
*   **ต้องปรับปรุง (0-4):** ไม่มีการเตรียมตัว, ตอบคำถามพื้นฐานไม่ได้

---

### **Milestone 2: Architectural Checkpoint (15%)**
**เป้าหมาย:** วัดความสามารถในการพิสูจน์สถาปัตยกรรมผ่าน Prototype และการออกแบบรายละเอียด

**1. การทำงานของ Prototype (40%)**
*   **ยอดเยี่ยม (9-10):** Live Demo ทำงานได้ครบ End-to-End อย่างราบรื่น, CI Pipeline ทำงานอัตโนมัติและเทสต์ผ่าน
*   **ดี (7-8):** Demo ส่วนใหญ่ทำงานได้ แต่อาจมีข้อผิดพลาดเล็กน้อย, CI Pipeline ถูกตั้งค่าแล้วแต่อาจยังไม่สมบูรณ์
*   **พอใช้ (5-6):** Demo ไม่สามารถทำงานได้ครบวงจร, ไม่มี CI Pipeline
*   **ต้องปรับปรุง (0-4):** ไม่มี Demo ที่ทำงานได้ หรือระบบล่มระหว่างนำเสนอ

**2. คุณภาพของการออกแบบและโค้ด (40%)**
*   **ยอดเยี่ยม (9-10):** Detailed Design สอดคล้องกับโค้ด, โครงสร้างโค้ดสะอาด มีการใช้ Design Principles ที่ชัดเจน, มี Unit Tests ที่มีความหมาย
*   **ดี (7-8):** Detailed Design ถูกต้องเป็นส่วนใหญ่, โครงสร้างโค้ดพอใช้ได้แต่ยังมี Code Smells, มี Unit Tests แต่ครอบคลุมไม่ครบถ้วน
*   **พอใช้ (5-6):** เอกสารออกแบบไม่สอดคล้องกับโค้ด, โครงสร้างไม่ดี (Logic ปนใน Controller), ไม่มี Unit Tests
*   **ต้องปรับปรุง (0-4):** ไม่มีเอกสารออกแบบ, โค้ดยุ่งเหยิง (Big Ball of Mud)

**3. การสาธิตและอธิบายเชิงเทคนิค (20%)**
*   **ยอดเยี่ยม (9-10):** สามารถอธิบาย "เหตุผล" เบื้องหลังการเลือกใช้เทคโนโลยี, pattern และโครงสร้างโค้ดได้อย่างมั่นใจและชัดเจน
*   **ดี (7-8):** สามารถอธิบายได้ว่า "โค้ดทำอะไร" แต่ยังอธิบาย "เหตุผล" ได้ไม่ลึกซึ้งนัก
*   **พอใช้ (5-6):** อธิบายการทำงานในภาพรวมได้ แต่ตอบคำถามเชิงเทคนิคในรายละเอียดไม่ได้
*   **ต้องปรับปรุง (0-4):** ไม่สามารถอธิบายโค้ดหรือการตัดสินใจทางเทคนิคของตนเองได้

---

### **Milestone 3: Final Product & Project Portfolio (15%)**
**เป้าหมาย:** วัดคุณภาพของผลิตภัณฑ์สุดท้าย, คุณภาพของกระบวนการทั้งหมด, และการเรียนรู้ของนิสิต

**1. ความสมบูรณ์และเสถียรภาพของผลิตภัณฑ์ (30%)**
*   **ยอดเยี่ยม (9-10):** ฟังก์ชันหลักทำงานได้ครบถ้วนและถูกต้อง, แอปพลิเคชันมีความเสถียร จัดการ Error Cases ได้ดี
*   **ดี (7-8):** ฟังก์ชันหลักส่วนใหญ่ทำงานได้ แต่อาจมี Bug หรือกรณีที่ยังจัดการไม่ได้บ้าง
*   **พอใช้ (5-6):** มีเพียงบางฟังก์ชันที่ทำงานได้, โปรแกรมมี Bug ค่อนข้างเยอะ
*   **ต้องปรับปรุง (0-4):** แอปพลิเคชันไม่สามารถทำงานได้ตามฟังก์ชันหลัก

**2. คุณภาพของการออกแบบและโค้ดโดยรวม (50%)**
*   **ยอดเยี่ยม (9-10):** โค้ดสะอาดสะท้อนความเข้าใจในหลักการออกแบบอย่างลึกซึ้ง, มีการใช้ Design Patterns อย่างสมเหตุสมผล, มีการทำ Refactoring เพื่อปรับปรุงคุณภาพโค้ดอย่างเห็นได้ชัด, `README.md` สมบูรณ์
*   **ดี (7-8):** โค้ดมีโครงสร้างที่ดี, การใช้ Pattern ถูกต้อง, ยังมี Bad Smells บางส่วน
*   **พอใช้ (5-6):** โค้ดทำงานได้แต่โครงสร้างไม่ดี ดูแลรักษายาก
*   **ต้องปรับปรุง (0-4):** โครงสร้างโค้ดยุ่งเหยิง ไม่มีการนำความรู้จากการเรียนมาประยุกต์ใช้

**3. การนำเสนอและสรุปการเรียนรู้ (20%)**
*   **ยอดเยี่ยม (9-10):** นำเสนอได้ดีเยี่ยม สรุปสิ่งที่ได้เรียนรู้ได้อย่างน่าสนใจ, Individual Reflection มีความลึกซึ้งและสะท้อนการคิดวิเคราะห์
*   **ดี (7-8):** การนำเสนอชัดเจน สรุปภาพรวมได้ดี, Reflection อธิบายสิ่งที่ทำไปได้แต่ยังขาดการวิเคราะห์
*   **พอใช้ (5-6):** การนำเสนอเป็นเพียงการ Demo โดยไม่มีบริบท, Reflection มีเนื้อหาผิวเผิน
*   **ต้องปรับปรุง (0-4):** การนำเสนอไม่น่าสนใจ, ไม่มี Reflection หรือมีแต่ไม่มีคุณภาพ
